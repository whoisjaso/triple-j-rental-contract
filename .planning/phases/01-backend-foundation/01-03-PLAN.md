  <files>src/lib/auth.tsx, src/pages/AdminLogin.tsx</files>
  <action>
  1. Create `src/lib/auth.tsx` with AuthProvider context:
     - Interface: `{ user: User | null, session: Session | null, loading: boolean, signIn: (email, password) => Promise<void>, signOut: () => Promise<void> }`
     - Import `User` and `Session` types from `@supabase/supabase-js`
     - Import `supabase` from `../lib/supabase`
     - In useEffect, call `supabase.auth.getSession()` to get initial session, then set user/session/loading
     - Subscribe to `supabase.auth.onAuthStateChange()` to keep auth state in sync
     - `signIn` calls `supabase.auth.signInWithPassword({ email, password })` and throws on error
     - `signOut` calls `supabase.auth.signOut()`
     - Export `AuthProvider` component and `useAuth` hook
     - CRITICAL: `loading` starts as `true` and is set to `false` only after `getSession()` resolves. This prevents flash of login page.

  2. Create `src/pages/AdminLogin.tsx`:
     - Simple form with email and password inputs
     - Use `useAuth()` hook to get `signIn` function and `user`
     - On form submit, call `signIn(email, password)` in a try/catch
     - Show error message on failed login (wrong credentials)
     - If user is already logged in, redirect to `/admin` using `Navigate` from react-router
     - Style with Tailwind v4 classes using the project's brand:
       - forest-green header text
       - gold accent border/button
       - Clean, centered card layout
       - Company name "Triple J Auto Investment LLC" at top
       - "Admin Login" heading
       - Email input, password input, submit button
       - Error message area (red text)
     - Input styling: use the same `legal-input` CSS class from main.css for consistent look, or use standard Tailwind border-b inputs
     - Button: `bg-forest-green text-white hover:bg-green-900` rounded, full width
     - Show loading spinner/text on the button while signIn is in progress (local useState for isSubmitting)
  </action>
