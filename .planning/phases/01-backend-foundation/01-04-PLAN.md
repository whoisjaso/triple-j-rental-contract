  <name>Task 1: Create Zustand store and agreement CRUD library</name>
  <files>src/stores/agreementStore.ts, src/lib/agreements.ts</files>
  <action>
  1. Create `src/stores/agreementStore.ts`:
     - Import `create` from `zustand`
     - Import `AgreementData` from `../types`
     - Define `AgreementStore` interface:
       ```typescript
       interface AgreementStore {
         data: AgreementData
         isDirty: boolean
         isSaving: boolean
         agreementId: string | null      // Supabase row id (null for new)
         agreementNumber: string | null   // TJ-YYYY-NNNN (null until saved)
         setData: (data: AgreementData) => void
         updateField: (section: keyof AgreementData, field: string, value: unknown) => void
         setAgreementMeta: (id: string, number: string) => void
         setSaving: (saving: boolean) => void
         reset: () => void
       }
       ```
     - Use `create<AgreementStore>()((set) => ({ ... }))` pattern (Zustand v5 TypeScript)
     - Copy the `initialData` object from the original App.tsx (lines 10-55) as the default state
     - `updateField` uses the same nested spread pattern as the original `update` function
     - `reset` returns state to initial values with null id/number
     - Export `useAgreementStore`

  2. Create `src/lib/agreements.ts`:
     - Import `supabase` from `./supabase`
     - Import `AgreementData` from `../types`
     - Import `Json` from `./database.types`

     Functions to implement:

     a. `createAgreement(data: AgreementData, userId: string): Promise<{ id: string, agreement_number: string }>`
        - Insert into `agreements` table: `{ data: data as unknown as Json, created_by: userId }`
        - Use `.select('id, agreement_number').single()` to get back the auto-generated agreement_number
        - After successful insert, write to `audit_log`:
          ```typescript
          await supabase.from('audit_log').insert({
            agreement_id: id,
            action: 'created',
            actor_type: 'admin',
            actor_id: userId,
            metadata: { agreement_number }
          })
          ```
        - Return `{ id, agreement_number }`
        - Throw on error

     b. `getAgreement(id: string): Promise<AgreementRow>`
        - Select from `agreements` where `id = id`, `.single()`
        - Return the row
        - Throw on error

     c. `updateAgreement(id: string, data: AgreementData, userId: string): Promise<void>`
        - Update `agreements` set `data = data as unknown as Json` where `id = id`
        - Write to `audit_log` with action 'updated'
        - Throw on error

     d. `listAgreements(): Promise<AgreementListItem[]>`
        - Define `AgreementListItem` type: `{ id: string, agreement_number: string, status: string, created_at: string, updated_at: string, renter_name: string | null }`
        - Select `id, agreement_number, status, created_at, updated_at, data` from agreements
        - Order by `created_at` descending
        - Map results to extract renter name from JSONB data: `(row.data as any)?.renter?.fullName || null`
        - Return the mapped list

     e. `getAuditLog(agreementId: string): Promise<AuditLogEntry[]>`
        - Select from `audit_log` where `agreement_id = agreementId`
        - Order by `created_at` ascending
        - Return entries

     Export all functions and the `AgreementListItem` type.

     IMPORTANT: The audit log insert uses the current authenticated user's ID. The IP address and user agent will be captured by the PostgreSQL helper functions where possible, or left null for now (application-side IP capture will be refined in Phase 2 when clients interact).
  </action>
  <verify>
  1. `npx tsc --noEmit` passes
  2. `src/stores/agreementStore.ts` exports `useAgreementStore` with initial AgreementData shape
  3. `src/lib/agreements.ts` exports createAgreement, getAgreement, updateAgreement, listAgreements, getAuditLog
  </verify>
  <done>Zustand store manages form state. CRUD library handles all database operations with audit logging.</done>
