**Create `supabase/functions/manheim-pipeline/parsers/ove.ts`:**

Import `DOMParser` from `jsr:@b-fuze/deno-dom`. Import `decodeGmailBase64` from `../gmail.ts`. Import `ParsedVehicle` from `../types.ts`.

Export function `parseOVEEmail(base64Body: string): ParsedVehicle`:
1. Decode the base64 body using `decodeGmailBase64`.
2. Parse HTML with `new DOMParser().parseFromString(html, "text/html")`.
3. OVE Purchase Confirmation emails have structured HTML with labeled fields. The email subject contains: "Manheim.com Purchase Confirmation (YEAR MAKE MODEL VIN SELLER)".
4. Extract fields by searching the HTML DOM for text patterns:
   - VIN: Look for text containing "VIN" label, extract the 17-char alphanumeric value
   - Year, Make, Model: From the vehicle description (e.g., "2019 Nissan Pathfinder SV Hard Top")
   - Mileage: Pattern like "113,377 mi"
   - Colors: "Black ext" / "GRY int" patterns
   - Purchase Amount: Dollar amount pattern "$2,500"
   - Buy Fee: Dollar amount after "Buy Fee" label
   - Total Amount: Dollar amount after "Total" label
   - Purchase Date: Date pattern "MM/DD/YY"
   - Pick-Up Location: City, State ZIP pattern
   - Seller: Text after "Seller" label
5. Use `querySelectorAll("td")` or similar to iterate table cells and find labeled values. OVE emails use HTML tables with label-value pairs.
6. Include a `parseSubjectLine(subject: string)` helper that extracts year, make, model, VIN from the subject pattern "Manheim.com Purchase Confirmation (YEAR MAKE MODEL VIN SELLER)".
7. Return a populated `ParsedVehicle` object. Use empty strings for fields that cannot be found (graceful degradation, not throws).

**Create `supabase/functions/manheim-pipeline/nhtsa.ts`:**

Import `NHTSAData` from `./types.ts`.

Export function `decodeVIN(vin: string): Promise<NHTSAData>`:
1. GET `https://vpic.nhtsa.dot.gov/api/vehicles/DecodeVin/${vin}?format=json`
2. Parse response: `data.Results` is an array of `{ Variable: string, Value: string }`.
3. Extract using the EXACT field names (verified from API docs):
   - `"Gross Vehicle Weight Rating From"` -> gvwr
   - `"Drive Type"` -> driveType
   - `"Displacement (L)"` -> engineDisplacement
   - `"Fuel Type - Primary"` -> fuelType
   - `"Body Class"` -> bodyClass
   - `"Doors"` -> doors
4. Return `NHTSAData`. Use empty strings for missing values.
5. Wrap in try/catch — on failure, return all-empty NHTSAData (per design doc: "NHTSA VIN decode fails -> Leave Empty Weight blank").

Export function `appendVehicleRow(token: string, values: string[][]): Promise<void>`:
1. POST to `https://sheets.googleapis.com/v4/spreadsheets/${SHEET_ID}/values/A1:append?valueInputOption=USER_ENTERED&insertDataOption=INSERT_ROWS`
2. Body: `{ values }`. MUST use `USER_ENTERED` (not RAW) so formulas and dates are interpreted.
3. Throw on non-200 with response details.

Export function `findRowByVIN(token: string, vin: string): Promise<number | null>`:
1. GET `https://sheets.googleapis.com/v4/spreadsheets/${SHEET_ID}/values/A:A`
2. Parse response: `data.values` is a 2D array. Each row is `[vinValue]`.
3. Find the row index where `row[0] === vin`.
4. Return `rowIndex + 1` (1-indexed for Sheets) or `null` if not found.

Export function `updateCells(token: string, range: string, values: string[][]): Promise<void>`:
1. PUT to `https://sheets.googleapis.com/v4/spreadsheets/${SHEET_ID}/values/${range}?valueInputOption=USER_ENTERED`
2. Body: `{ values }`.
3. Throw on non-200.

Export function `buildVehicleRow(vehicle: ParsedVehicle, nhtsa: NHTSAData): string[]`:
This builds the 40-column row array per the design doc column mapping:
- Column A (VIN): `vehicle.vin`
- Column B (search_key): `"${vehicle.year} ${vehicle.make} ${vehicle.model}"`
- Column C (Year): `vehicle.year`
- Column D (Make): `vehicle.make`
- Column E (Model): `vehicle.model`
- Column F (Exterior/Interior Color): `"${vehicle.exteriorColor} / ${vehicle.interiorColor}"`
- Column G (Empty Weight): `nhtsa.gvwr`
- Column H (Odometer): `vehicle.odometer`
- Column I (Condition): `"Pending Inspection"` (default — updated later by Sale Docs parser)
- Column J (Title Type): `"Clean"` (default — updated later by Sale Docs parser)
- Column K (Date Acquired): `vehicle.purchaseDate`
- Column L (Date Listed): `vehicle.purchaseDate` (same as acquired)
- Column M (Days in Stock): `=TODAY()-K{ROW}` (formula — will need row number)
- Column N (Purchase Price): `vehicle.purchasePrice`
- Column O (Towing Cost): `""` (blank — updated by Central Dispatch parser or defaults to $700)
- Column P (Mechanical Cost): `"0"`
- Column Q (Cosmetic Cost): `"0"`
- Column R (Other Costs): `vehicle.buyFee`
- Column S (All-In Cost): `=N{ROW}+O{ROW}+P{ROW}+Q{ROW}+R{ROW}` (formula)
- Column T (Target Margin %): `"0.25"`
- Column U (Target List Price): `=S{ROW}*(1+T{ROW})` (formula)
- Column V (Floor Margin %): `"0.10"`
- Column W (Floor Price): `=S{ROW}*(1+V{ROW})` (formula)
- Column X (Photo Link): `""` (blank — filled by photo upload page)
- Column Y (Listing Status): `"Incoming"` (per user decision — NOT "Available")
- Column Z (Current List Price): `=U{ROW}` (formula — equals Target List Price initially)
- Columns AA-AN: All blank `""` (14 post-sale columns)

NOTE about formulas: Since Sheets `append` adds to the end and we use `USER_ENTERED`, formulas like `=TODAY()-K2` need the actual row number. Two approaches: (1) Use fixed formulas referencing the same row with `INDIRECT("K"&ROW())` or just `=TODAY()-INDIRECT("K"&ROW())`, or (2) pass the formulas without row numbers and let Sheets auto-resolve since `append` places at the next row. The simplest approach: use `INDIRECT("col"&ROW())` pattern so formulas work regardless of which row they land in. Example: `=TODAY()-INDIRECT("K"&ROW())` for Days in Stock.
