**Create `src/components/photo/CameraCapture.tsx`:**

This component opens the camera, shows a live viewfinder, and captures a photo on tap.

Props:
- `shotName: string` — current shot label (e.g., "Front", "Rear", "Driver Side")
- `shotIndex: number` — current shot number (0-6)
- `totalShots: number` — always 7
- `onCapture: (blob: Blob) => void` — called with captured photo blob
- `onSkip?: () => void` — optional skip for shots that are impossible (e.g., trunk on pickup truck)

1. Use `navigator.mediaDevices.getUserMedia({ video: { facingMode: "environment", width: { ideal: 1920 }, height: { ideal: 1080 } } })` to open the rear camera.
2. Display the video stream in a `<video>` element that fills the viewport. Autoplay, playsInline, muted attributes required for mobile.
3. Show an overlay with:
   - Top bar: Shot name label (e.g., "FRONT VIEW") and progress (e.g., "1 of 7")
   - Visual guide hint per shot type (e.g., "Center the front of the vehicle" for front shot)
   - Bottom: Large circular capture button (white circle with border, like iOS camera)
   - "Skip" text button below capture button
4. On capture button tap:
   a. Create a canvas element matching video dimensions.
   b. Draw the video frame to canvas: `ctx.drawImage(videoEl, 0, 0)`.
   c. Convert to blob: `canvas.toBlob(callback, "image/jpeg", 0.85)`.
   d. Compress with `browser-image-compression`:
      ```
      const file = new File([blob], `${shotName}.jpg`, { type: "image/jpeg" });
      const compressed = await imageCompression(file, {
        maxSizeMB: 1,
        maxWidthOrHeight: 1920,
        useWebWorker: true,
      });
      ```
   e. Call `onCapture(compressed)` with the compressed blob.
5. On unmount, stop all video tracks: `stream.getTracks().forEach(t => t.stop())`.
6. Handle getUserMedia errors gracefully: if camera access denied, show fallback with `<input type="file" accept="image/*" capture="environment">` for single-shot gallery/camera picker.
7. Style: fullscreen camera viewfinder experience. Dark background. Capture button prominent and centered at bottom.

The 7 shot types (in order):
1. Front
2. Rear
3. Driver Side
4. Passenger Side
5. Interior
6. Dashboard
7. Trunk

Each shot has a brief instruction:
- Front: "Center the front of the vehicle, include headlights"
- Rear: "Center the rear, include taillights and plate area"
- Driver Side: "Full driver side profile"
- Passenger Side: "Full passenger side profile"
- Interior: "Front seats and center console"
- Dashboard: "Dashboard, steering wheel, odometer"
- Trunk: "Open trunk or cargo area"

**Create `src/components/photo/PhotoReview.tsx`:**

This component shows all captured photos in a grid, uploads them to Supabase Storage, and triggers auto-publish when complete.

Props:
- `photos: { shotName: string, blob: Blob }[]` — array of captured photos
- `vehicle: { vin: string, year: string, make: string, model: string }` — selected vehicle
- `onComplete: () => void` — called after successful upload and publish
- `onRetake: (index: number) => void` — go back to camera for a specific shot

1. Display photos in a 2-column grid. Each thumbnail shows the shot name label and a "Retake" button.
2. Show "Upload All Photos" button at bottom (large, prominent, brand green).
3. On upload:
   a. Create Supabase Storage bucket reference: `supabase.storage.from("vehicle-photos")`. The bucket must be public (set up in Plan 01 migration or manually).
   b. For each photo, upload to path `{VIN}/{shotName}-{Date.now()}.jpg`:
      ```
      const path = `${vehicle.vin}/${photo.shotName}-${Date.now()}.jpg`;
      await supabase.storage.from("vehicle-photos").upload(path, photo.blob, { contentType: "image/jpeg" });
      ```
   c. Get public URL for each uploaded photo:
      ```
      const { data: { publicUrl } } = supabase.storage.from("vehicle-photos").getPublicUrl(path);
      ```
   d. Show upload progress: update each photo's status (pending -> uploading -> done) with a progress indicator.
4. After all photos uploaded:
   a. Collect all public URLs into a comma-separated string.
   b. Update the vehicle's Photo Link in the Google Sheet. Since the photo page is client-side and cannot call Google Sheets API directly (OAuth credentials), use a small Supabase Edge Function or update the vehicles table in Supabase directly. The simplest approach: update the Supabase `vehicles` table directly:
      - `supabase.from('vehicles').update({ gallery: urls, status: 'Available' }).eq('vin', vehicle.vin)`
      - The gallery field likely stores photo URLs; status flip to 'Available' makes it live
   c. Auto-publish: flip status from "Incoming" to "Available" in the same update.
   d. NOTE: The Google Sheet's Photo Link column will need to be synced separately. Since the Sheet is the source of truth for most data, but the website reads from Supabase, updating Supabase directly for photos and status is the pragmatic approach. If the existing sync is bi-directional for these columns, it will propagate. If not, the Sheet column stays blank (acceptable — photos are served from Supabase Storage).
5. Show success state: "Vehicle published! Now live on website." with a checkmark animation.
6. Call `onComplete()`.

**Create `src/pages/PhotoUpload.tsx`:**

Standalone page at /photos — the main orchestrator for the photo workflow.

1. State management (useState):
   - `step`: 'select' | 'capture' | 'review' | 'complete'
   - `selectedVehicle`: vehicle object or null
   - `photos`: array of { shotName, blob } (accumulates during capture)
   - `currentShotIndex`: 0-6
2. Define SHOT_SEQUENCE constant: array of 7 shot objects with `name` and `instruction` strings.
3. Flow:
   - **Step 'select'**: Render `<VehicleSelector onSelect={...} />`. On select, set vehicle and transition to 'capture'.
   - **Step 'capture'**: Render `<CameraCapture>` with current shot name/index. On capture, add photo to array, advance to next shot. After 7th shot (or skip), transition to 'review'.
   - **Step 'review'**: Render `<PhotoReview>` with all photos and vehicle. On retake, go back to capture for that shot. On complete, transition to 'complete'.
   - **Step 'complete'**: Show success message with vehicle info. "Back to vehicle list" button resets to 'select'.
4. Header: Show "Triple J Photo Upload" and current vehicle info (when selected). Keep it minimal — fast loading, no heavy chrome.
5. Style: mobile-first, full-screen camera during capture, clean white background for select/review. Use Tailwind. Brand green accents.
6. The page does NOT require authentication — it is bookmarkable and fast-loading. However, Supabase Storage uploads require the anon key (public bucket). If the vehicle-photos bucket requires auth, use the existing supabase client from `src/lib/supabase.ts`.
7. Handle browser back button: in capture mode, back goes to previous shot (not browser back). Use window.history or prevent default.
