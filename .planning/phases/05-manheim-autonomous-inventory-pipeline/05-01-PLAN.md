<task type="auto">
  <name>Task 2: Edge Function entry point with OAuth2 and Gmail client</name>
  <files>supabase/functions/manheim-pipeline/index.ts, supabase/functions/manheim-pipeline/gmail.ts</files>
  <action>
Create `supabase/functions/manheim-pipeline/gmail.ts` — the Gmail API client module:

1. `getAccessToken()`: Exchange refresh token for access token via POST to `https://oauth2.googleapis.com/token`. Read GOOGLE_REFRESH_TOKEN, GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET from `Deno.env.get()`. Return the access_token string. Throw on non-200 response with error details.

2. `decodeGmailBase64(b64: string): string`: Convert URL-safe base64 to standard base64 (replace `-` with `+`, `_` with `/`) then `atob()`. This is CRITICAL — Gmail returns URL-safe base64 per RFC 4648.

3. `decodeGmailBase64ToBytes(b64: string): Uint8Array`: Same conversion but returns bytes. Needed for PDF attachment decoding.

4. `listUnreadMessages(token: string, query: string): Promise<string[]>`: GET `https://gmail.googleapis.com/gmail/v1/users/me/messages?q={query}&maxResults=10`. Return array of message IDs. Return empty array if no messages.

5. `getMessage(token: string, messageId: string): Promise<GmailMessage>`: GET message with `format=full`. Return typed GmailMessage.

6. `getAttachment(token: string, messageId: string, attachmentId: string): Promise<Uint8Array>`: GET attachment data, decode URL-safe base64 to bytes using `decodeGmailBase64ToBytes`.

7. `markAsRead(token: string, messageId: string): Promise<void>`: POST to modify endpoint, remove UNREAD label.

8. `getMessageBody(message: GmailMessage): string`: Walk message.payload.parts recursively to find `text/html` or `text/plain` part. Decode its base64 body data. Return the decoded string. Handle both single-part and multipart messages.

Import types from `./types.ts`. All functions use pure `fetch()` — no SDK. Export all functions.

Create `supabase/functions/manheim-pipeline/index.ts` — the Edge Function entry point:

1. Import `Deno.serve` pattern (standard Supabase Edge Function).
2. Import `createClient` from `npm:@supabase/supabase-js@2.97.0` for pipeline_log writes.
3. Import gmail functions from `./gmail.ts`.
4. Add pipeline secret check: read `x-pipeline-secret` header, compare against `Deno.env.get("PIPELINE_SECRET")`. Return 401 if mismatch. This prevents unauthorized invocations (per Pitfall 8 from RESEARCH.md).
5. Create Supabase client using `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` (both auto-available in Edge Functions).
6. Main handler: try/catch wrapper. On entry, call `getAccessToken()`. Then process three email types sequentially:
   - OVE Purchase Confirmations (query: `from:support@ove.com subject:"Purchase Confirmation" is:unread`)
   - Central Dispatch ACCEPTED (query: `from:do-not-reply@centraldispatch.com subject:"has been ACCEPTED" is:unread`)
   - Manheim Sale Documents (query: `from:noreply@manheim.com subject:"Sale Documents" is:unread`)
7. For now, stub the processing functions as TODO comments — they will be implemented in Plans 02 and 03. The structure should be:
   ```
   const oveIds = await listUnreadMessages(token, OVE_QUERY);
   for (const id of oveIds) { /* TODO: processOVEEmail(token, supabase, id) */ }
   ```
8. Return JSON response: `{ processed: { ove: N, cd: N, saleDocs: N }, errors: [] }`.
9. On catch, log error to pipeline_log and return 500.
10. Add CORS headers for Supabase Edge Function standard pattern.
