**Create `supabase/functions/manheim-pipeline/parsers/central-dispatch.ts`:**

Import `CentralDispatchData` from `../types.ts`. Import `decodeGmailBase64` from `../gmail.ts`.

Central Dispatch ACCEPTED emails are plain text (NOT HTML). The email body contains labeled fields. Use regex parsing, not DOM.

Export function `parseCentralDispatchEmail(base64Body: string): CentralDispatchData`:
1. Decode the base64 body with `decodeGmailBase64`.
2. Extract using regex patterns against the plain text body:
   - **Load ID**: Pattern like `Load ID: 1234567` or from the subject line "Load ID: XXXXXXX has been ACCEPTED by CARRIER"
   - **Price**: Pattern like `Price: $750.00` or `Total Price: $750.00`. This is the LOAD price (not per-vehicle).
   - **Number of Vehicles**: Count VIN entries or look for explicit count field.
   - **VINs**: Each vehicle listed as "YEAR MAKE MODEL" followed by a VIN (17-char alphanumeric). Collect all VINs.
   - **Per-vehicle towing cost**: `loadPrice / vehicleCount`. This is the number written to each vehicle's Towing Cost column.
3. Handle edge cases:
   - Load price may have commas: "$1,250.00" — strip to number.
   - VINs may appear on separate lines or in a list format.
   - Vehicle count may be 1 (single vehicle shipment) or more.
4. Return `CentralDispatchData` with loadId, vins array, loadPrice (as string), vehicleCount, perVehicleCost (as number).

Also export a helper `parseSubjectForLoadId(subject: string): string | null`:
- Extract Load ID from subject: "Load ID: XXXXXXX has been ACCEPTED by CARRIER_NAME"
- Return the Load ID string or null.

Per RESEARCH.md Pitfall 5: Multiple ACCEPTED emails may arrive for the same Load ID (carriers accept/cancel). The pipeline should process each one — the most recent ACCEPTED updates the towing cost. The pipeline_log tracks Load IDs via metadata for debugging.

**Create `supabase/functions/manheim-pipeline/parsers/sale-docs.ts`:**

Import `extractText, getDocumentProxy` from `npm:unpdf`. Import `SaleDocData` from `../types.ts`. Import `decodeGmailBase64`, `getAttachment`, `decodeGmailBase64ToBytes` from `../gmail.ts`. Import `GmailMessage` from `../types.ts`.

Export function `parseSaleDocsEmail(token: string, message: GmailMessage): Promise<SaleDocData[]>`:
1. Extract VIN(s) from the email body text (the body may reference one or more VINs).
2. Walk `message.payload.parts` recursively to find the PDF attachment:
   - Look for a part with `mimeType === "application/pdf"` or `"application/octet-stream"`.
   - The filename pattern is `MM-DD-YYYY_ACCOUNT_Vehicle_Sale_Documents.pdf`.
   - Get the `attachmentId` from `part.body.attachmentId`.
3. Download the PDF via `getAttachment(token, message.id, attachmentId)`.
4. Extract text using unpdf:
   ```
   const pdf = await getDocumentProxy(pdfBytes);
   const { text } = await extractText(pdf, { mergePages: true });
   ```
5. Parse the extracted text for:
   - **Condition**: Regex for "Condition" label followed by a value (e.g., "Good", "Fair", "Average"). Use broad matching since PDF layout varies.
   - **Title Type**: Regex for "Title" label followed by type (e.g., "Clean", "Salvage", "Rebuilt").
6. Return array of `SaleDocData` (one per VIN found in the email). For each: `{ vin, condition, titleType }`.
7. Fallbacks per design doc:
   - Condition not found: `"Pending Inspection"`
   - Title Type not found: `"Clean"`
8. If PDF parsing fails entirely (unpdf error), catch and return fallback values for all VINs found in body.

**Update `supabase/functions/manheim-pipeline/index.ts`:**

Replace the Central Dispatch and Sale Docs TODO stubs with real processing functions:

**`processCentralDispatchEmails(token: string, supabase: SupabaseClient)`:**
1. List unread CD emails: `from:do-not-reply@centraldispatch.com subject:"has been ACCEPTED" is:unread`
2. For each message:
   a. Get full message.
   b. Extract body, parse with `parseCentralDispatchEmail`.
   c. For each VIN in the parsed data:
      - Call `findRowByVIN(token, vin)` to get the row number.
      - If row found: `updateCells(token, "O{row}", [[perVehicleCost.toString()]])` (column O = Towing Cost).
      - If row NOT found: Log as 'skipped' (vehicle not yet in sheet — maybe OVE email hasn't been processed yet).
   d. Log to pipeline_log: status 'processed', metadata includes loadId and perVehicleCost.
   e. Mark email as read.
   f. NO SMS for Central Dispatch updates (per user decision: "silent").
3. Wrap each email in try/catch. On error: log to pipeline_log with status 'error'. Do NOT mark as read.

**`processSaleDocsEmails(token: string, supabase: SupabaseClient)`:**
1. List unread Sale Docs emails: `from:noreply@manheim.com subject:"Sale Documents" is:unread`
2. For each message:
   a. Get full message.
   b. Call `parseSaleDocsEmail(token, message)` — this downloads and parses the PDF.
   c. For each VIN/data pair returned:
      - Call `findRowByVIN(token, vin)`.
      - If found: Update column I (Condition) and column J (Title Type) using `updateCells`.
      - If not found: Log as 'skipped'.
   d. Log to pipeline_log: status 'processed'.
   e. Mark email as read.
   f. NO SMS for Sale Docs updates (per user decision: "silent").
3. Wrap each email in try/catch. On error: log to pipeline_log with status 'error'. Do NOT mark as read.

Update the main handler to call all three processors sequentially:
